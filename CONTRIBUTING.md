# Contributing to Node-RED Vikunja Tasks

Thank you for your interest in contributing to Node-RED Vikunja Tasks! This document provides guidelines for contributing.

## Getting Started

1. Fork the repository
2. Create a branch for your feature or bug fix
3. Make your changes
4. Test your changes
5. Submit a pull request

## Development Setup

```bash
git clone https://github.com/blaxer/node-red-contrib-workspace-vikunja.git
cd node-red-contrib-workspace-vikunja
npm install
```

## Development Guidelines

- Follow the existing code style
- Add comments for complex logic
- Update documentation for new features
- Test your changes with Node-RED

## Pull Request Process

1. Update the README.md with details of changes if needed
2. Update the CHANGELOG.md with your changes
3. The PR will be merged once you have the sign-off of a maintainer

## Code Style

- Use 4 spaces for indentation
- Use double quotes for strings
- Add semicolons at the end of statements
- Follow Node-RED node development best practices

## Questions?

Open an issue or contact the maintainer.
